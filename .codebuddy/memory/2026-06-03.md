## start_python.py 闪退修复
- **根因**: 代码中使用了 Python 3.10+ 语法（`int | None`、`tuple[...]` 泛型内置类型），如果用户环境是 Python 3.8/3.9，导入阶段就会 `TypeError` 导致窗口一闪而过。
- **修复内容**:
  - `start_python.py`: 顶层添加依赖包前置检查 + `try/except` 捕获导入异常并 `input()` 暂停，避免双击运行时闪退。
  - `backend/utils.py`: `int | None` → `Optional[int]`；添加 `from typing import Optional`。
  - `backend/excel_handler.py`: `tuple[...]` → `Tuple[...]`；导入 `Tuple`。
  - `backend/crud.py`: `tuple[...]` → `Tuple[...]`；导入 `Tuple`。

## 登录 500 错误排查

## 服务器端口占用启动失败修复 (Errno 10048)
- **症状**: `start_server()` 启动时报 `[Errno 10048] 端口 5002 已被占用`，但 `is_server_running()` 检测时未发现
- **根因**: 端口可能被残留进程或 TIME_WAIT 状态占用，启动前检测漏掉了
- **修复内容**:
  - 新增 `_force_free_port(port)` 函数：先尝试 `stop_server()` → 再用 `netstat -ano` + `taskkill /F` 强杀
  - `start_server()`: 启动失败后自动释放端口并重试（最多2次），重试前打印日志最后几条 ERROR 行便于排查
  - `restart_server()`: 停止后主动调用 `_force_free_port()` 确保端口完全释放，同样支持重试
  - `_start_server_background()`: 启动前清空旧日志文件，避免误读上次的错误信息
