# 2025-06-03 工作日志

## 导出Excel找不到文件 + 证书下载页面修复

### 问题1：局域网用户导出Excel找不到文件
- **原因**：`saveFileWithPicker` 使用 `window.showSaveFilePicker` API，该API仅在**安全上下文（受信任的HTTPS）**下可用。未安装CA证书时浏览器标记为"不安全"，API被禁用，回退到 `a.download` 方式，文件下载到浏览器默认"下载"文件夹而非桌面。
- **修复**：
  - `saveFileWithPicker` 增加 `window.isSecureContext` 检查，仅安全上下文才使用文件选择器
  - 返回 `usedPicker` 标志区分两种下载方式
  - `exportData()` 根据 `usedPicker` 显示不同提示：
    - 使用选择器："✓ 已保存到桌面！"
    - 回退下载："✓ 已导出到浏览器下载文件夹（Ctrl+J 查看），建议安装证书后可选择保存位置"
- 文件：`static/HJ.html`

### 问题2：根路径改为证书下载页面
- **修复**：`main.py` 中 `/` 路由返回 `cert-download.html`，系统主页面通过 `/HJ.html` 访问
- 文件：`main.py`

### 问题3：更新证书下载页面
- 添加醒目提示：未安装证书时导出Excel只能下载到默认文件夹
- 添加备用下载链接 `/ca.crt`
- 修复过时的 `node regenerate-cert.js` 引用为 `python generate-cert-python.py`
- 修复 BAT_CONTENT 中的固定IP为通用文字
- 文件：`static/cert-download.html`

## 用户认证系统

### 后端
- 新建 `backend/auth.py`：JWT Token 认证（python-jose + passlib）
  - `create_access_token` / `decode_token` / `verify_password` / `get_password_hash`
  - `get_current_user`（可选认证，未登录返回 None）
  - `require_auth`（强制认证，未登录 401）
  - `require_admin`（管理员权限检查，非管理员 403）
  - `init_default_admin`：首次启动自动创建默认管理员 `admin/admin`
- 更新 `backend/models.py`：User 添加 `display_name`、`index=True`
- 更新 `backend/crud.py`：添加用户 CRUD（get_user_by_username, create_user, update_user_password, update_user_last_login, get_all_users, delete_user）
- 更新 `backend/schemas.py`：添加 UserCreate, UserResponse, UserLoginRequest, UserLoginResponse
- 更新 `backend/database.py`：`init_db()` 时调用 `init_default_admin()`
- 更新 `main.py`：新增路由
  - `POST /api/login` — 登录返回 JWT
  - `POST /api/logout` — 登出
  - `GET /api/user` — 获取当前用户信息（未登录返回 logged_in: false）
  - `PUT /api/user/password` — 修改密码（需认证）
  - `GET /login.html` — 登录页面
- 文件：`backend/auth.py`, `backend/models.py`, `backend/crud.py`, `backend/schemas.py`, `backend/database.py`, `main.py`

### 前端
- 新建 `static/login.html`：登录页面，默认账号提示，登录成功后存储 token 并跳转
- 更新 `static/HJ.html`：
  - 页面加载时 `checkLogin()` 检查登录状态，未登录跳转 `/login.html`
