## 修复导入 BadZipFile / File is not a zip file 错误（2026-06-01 第四次）
- **根本原因**：用户上传的 Excel 文件实际是 `.xls`（Excel 97-2003 OLE2 格式，magic bytes: D0CF11E0），但文件扩展名是 `.xlsx`。代码仅根据扩展名选择 openpyxl 读取，导致 `BadZipFile` 错误
- **backend/excel_handler.py**：
  - 新增 `_detect_excel_format()` 函数：读取文件头 magic bytes 自动识别真实格式
    - PK 开头 → ZIP/XLSX 格式 → 用 openpyxl
    - D0CF11E0 开头 → OLE2/XLS 格式 → 用 xlrd
    - 其他 → 报错提示非 Excel 文件
  - `import_from_excel()` 改为基于 magic bytes 检测格式而非文件扩展名
- **requirements.txt**：已添加 xlrd==2.0.1 依赖
- **测试验证**：用之前失败的调试文件测试，成功自动检测为 .xls 并用 xlrd 读取了 3677 行数据
